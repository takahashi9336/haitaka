# live_trip 変更履歴・設計決定録 (ADR)

## 2026-05-03 一覧UI・複数会場・しおり導線・移動タブ（設計書追随）

* **変更内容（実装→文書反映）**:
  - **遠征一覧**: KPI3枚、中央寄せ最大幅、期間トグルの初期「今後」＋クライアント側表示切替、「すべて」時の帯見出し、カードレイアウト／アクセント線／ソート解釈（今日に近い順）、イベント未紐付時の基準日（メイン目的地）を UI 設計書に明記。
  - **複数イベント・会場**: `enrichTripWithEvents` の `event_place` を全会場名の `・` 結合に変更。**詳細の目的地タブ**では会場カードをイベント由来で複数表示（重複まとめ・会場ごと Geocode 可）。**一覧 SQL（`getMyTrips`）の `event_place` は先頭イベント由来のまま**である旨をドメイン定義書に記載（一覧と詳細で文言ソースが異なる場合あり）。
  - **しおり**: `shiori.php` は維持。一覧・詳細からの **しおり導線ボタンは廃止**（URL直指定）を機能概要・機能一覧・画面遷移・UI設計に反映。
  - **移動タブ**: KPI、2カラム、日付グループ、フォームアコーディオン、一覧上の交通機関の色分け、自宅登録位置などを UI 設計書に追記。
  - **I-O**: `index` の `period` 既定値とクライアント切替、`show` の `enrichTripWithEvents` と目的地複数会場を追記。
* **背景・理由**: 実装が先行したため、設計書を現状の振る舞いと整合させる。
* **見送った代替案**: 一覧 `getMyTrips` の `event_place` サブクエリを詳細と同じ複数結合に揃える案は、SQL 負荷・行幅の検討が必要なため本 ADR ではドキュメント上の差分明示に留めた。

## 2026-04-29 設計書初版（docs 体系整備）

* **変更内容**: `docs/live_trip/` に基本設計・詳細設計・本 ADR を追加。実装は `private/apps/LiveTrip` / `www/live_trip` を根拠とした。
* **背景・理由**: AI・開発者が同一リファレンスを共有するため。旧 `docs/99_old/遠征管理_DB設計_ER図.md` の思想を最新スキーマに合わせて再記述。
* **見送った代替案**: コントローラと 1:1 の巨大 I-O 表（全メソッドのパラメータ列挙）はメンテコストが高いため、一覧表＋代表フローに留めた。

## 2026-04-29 遠征タイトル導入とイベント後追い紐付け

* **変更内容**: `lt_trip_plans.title` を導入し、遠征作成時の必須入力をイベント選択からタイトル入力へ変更。イベント紐付けは任意（0件可）とし、作成後に編集画面から追加可能にした。
* **背景・理由**: 「イベントを先に作らないと遠征管理を開始できない」UX課題を解消し、遠征を独立プロジェクトとして管理するため。
* **見送った代替案**: `lt_trip_plan_events` 側に擬似タイトルイベント（プレースホルダ）を登録する案は、データ責務が曖昧になるため不採用。

## 2026-04-29 タイムラインタブUI/UX改善（KPI/日付フィルタ/日表示刷新）

* **変更内容**:
  - `show.php` のタイムラインタブに、目的地/費用タブと統一感のある **ヘッダーKPIカード**（件数・日数・カテゴリ別バッジ、活動時間、表示切替）を新設。
  - KPIと追加フォームの間に **日付フィルタカード** を追加し、日表示/リスト表示を対象日付に絞り込み可能にした（全日表示も提供）。
  - **予定を追加カード** をアコーディオン化し、カテゴリトグル＋2行レイアウトへ整理。
  - **日表示（カレンダー）**を時間レーン型から「縦軸＋カード型」へ刷新し、空き時間に「⋯ 空き時間（クリックで追加）」要素を挿入。
  - **リスト表示**はサマリタブの行程プレビュー（リストUI）を踏襲。
* **背景・理由**: 当日の行動を「時系列で迷わず把握できる」ことを最優先にし、入力導線を折りたたみで省スペース化しつつ、日付単位での閲覧体験を強化するため。
* **制約・備考**: `lt_timeline_items` にカテゴリ列は持たないため、当面は `label` に絵文字プレフィックスを付与してカテゴリ表現を行う（必要なら将来的にカラム追加を検討）。
