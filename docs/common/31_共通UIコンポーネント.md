# 共通UIコンポーネント設計書

実装基点: [`private/components/`](../../private/components/)  
各画面では `require_once` で取り込み、グローバル変数やセッション（`$_SESSION['user']` 等）に依存するコンポーネントがある。

## 1. 概要

ログイン後のレイアウトで共有されるパーツ。多くは **Tailwind CSS**（CDN またはページ内での読込）および **Font Awesome 6.x** と併用する。

## 2. コンポーネント一覧

### sidebar.php — 共通サイドバー

| 項目 | 内容 |
|------|------|
| **ファイル** | [`private/components/sidebar.php`](../../private/components/sidebar.php) |
| **役割** | `$_SESSION['user']['apps']` のツリーからメニューを動的生成。現在 URI と `route_prefix` / `default_route` でアクティブ判定。モバイル幅では開閉用の UX を内包（`THEME_ALLOWED_TW` によるテーマ、`theme_primary` が hex の場合はインラインスタイル） |
| **呼び出し例** | `require_once __DIR__ . '/../private/components/sidebar.php';`（パスは呼び元に合わせる） |
| **主な依存** | セッションユーザ、`REQUEST_URI`。定数 `THEME_ALLOWED_TW`（未_define 時コンポーネント内で定義） |

### head_favicon.php — Favicon・共通CSS

| 項目 | 内容 |
|------|------|
| **ファイル** | [`private/components/head_favicon.php`](../../private/components/head_favicon.php) |
| **役割** | SVGデータURIのファビコン、Apple Touch Icon、`/assets/css/common.css` を `<head>` に出力 |
| **呼び出し例** | `<?php require_once __DIR__ . '/../../../components/head_favicon.php'; ?>`

### flash_toast.php — フラッシュトースト（サーバセッション）

| 項目 | 内容 |
|------|------|
| **ファイル** | [`private/components/flash_toast.php`](../../private/components/flash_toast.php) |
| **役割** | `$_SESSION['flash_success']` / `flash_error` を一度だけ表示して `unset`。右上固定のトーストDOM＋短命アニメーション |
| **注意** | クライアントから任意表示する **`App.toast`**（[`www/assets/js/core.js`](../../www/assets/js/core.js)）とは別系統 |

### theme_from_session.php — アプリテーマ変数の解決

| 項目 | 内容 |
|------|------|
| **ファイル** | [`private/components/theme_from_session.php`](../../private/components/theme_from_session.php) |
| **役割** | `$_SESSION['user']['apps']` から `app_key` に一致する `theme_primary` / `theme_light` を探し、`$themePrimary`、`$bodyBgClass`/`$bodyStyle`、`$headerIconBg`/`$headerIconStyle`、`$card*` 等をセット。関数 `findAppInSessionTree`, `getThemeVarsForApp` を提供 |
| **呼び出し例** | 画面側で `$appKey = 'hinata';` 等をセットしてから `require`。**インクルード前後で `$appKey` が必要**（ファイル先頭コメント参照） |

### guide_display.php — 共通ガイド（?アイコン＋モーダル）

| 項目 | 内容 |
|------|------|
| **ファイル** | [`private/components/guide_display.php`](../../private/components/guide_display.php) |
| **役割** | `$guideKey` 必須。`GuideDisplay` で `/api/guide.php?guide_key=` から本文取得。初回自動表示は `localStorage` |
| **呼び出し例** | `$guideKey = 'meetgreet_import'; require_once __DIR__ . '/guide_display.php';` |

### video_modal.php — 動画モーダル

| **ファイル** | [`private/components/video_modal.php`](../../private/components/video_modal.php) |
| **役割** | サイト内共通の動画埋め込みモーダル（詳細は呼び元画面に依存） |

### blog_image_modal.php — 画像モーダル

| **ファイル** | [`private/components/blog_image_modal.php`](../../private/components/blog_image_modal.php) |
| **役割** | ブログ／記事系で利用する画像拡大モーダル |

## 3. レイアウト典型パターン

多くの画面は次のような骨格となる。

```html
<div class="main-container">
  <!-- sidebar.php -->
  <div class="content-wrapper">
    <!-- 画面固有コンテンツ -->
  </div>
</div>
```

`common.css` では `--sidebar-width`、`body` のベース背景、`.main-container` / `.content-wrapper` のフレックス配置を定義している。
