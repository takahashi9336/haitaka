# haitaka（MyPlatform）システム構成と機能一覧

**目的**: 本リポジトリの全体像（技術スタック、ディレクトリ、認証・権限、主要モジュール）を1ファイルで把握するためのサマリーです。  
**補足**: 日向坂ポータル単体の詳細は [日向坂ポータル_システム構成書.md](./日向坂ポータル_システム構成書.md)、アプリ／ロールのDB設計は [設計_アプリ・ロール管理テーブル.md](./設計_アプリ・ロール管理テーブル.md) を参照してください。

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| 名称 | **haitaka**（Composer: `haitaka/integrated-platform`） |
| 説明 | 個人向け統合ポータル（タスク、メモ、エンタメ管理、日向坂関連コンテンツ等） |
| 公開ドキュメント | ルート `README.md` は「myportal」程度の短い説明のみ |

---

## 2. 技術スタック

| 層 | 採用技術 |
|----|----------|
| バックエンド | **PHP 8.2+**（PSR-4 オートロード、`Core\` / `App\` 名前空間） |
| データベース | **MySQL / MariaDB**（PDO、`private/.env` の `DB_*` で接続） |
| セッション | カスタム `SessionManager`（DB 永続化。Cookie は Secure / HttpOnly / SameSite=Lax、長期セッション設定あり） |
| フロントエンド | **Tailwind CSS**（CDN）、**Font Awesome**、バニラ JavaScript（`/www/assets/js/core.js` 等） |
| 依存管理 | **Composer**（`private/composer.json`）。フロント用 `package.json` はなし |

---

## 3. ディレクトリ構成（要点）

```
haitaka/
├── www/                    # Web ドキュメントルート（公開 URL に対応）
│   ├── index.php           # ダッシュボード（ログイン後ホームの一つ）
│   ├── login.php / logout.php
│   ├── admin/              # 管理画面エントリ
│   ├── hinata/             # 日向坂ポータル
│   ├── task_manager/       # タスク管理
│   ├── note/               # メモ
│   ├── focus_note/         # Focus Note（週次・月次・ゴール等）
│   ├── movie/              # 映画（TMDB 連携）
│   ├── drama/              # ドラマ（TMDB）
│   ├── anime/              # アニメ（Annict OAuth）
│   ├── live_trip/          # ライブ遠征・旅程
│   ├── entame/             # エンタメ視聴統計ダッシュボード
│   ├── health/             # ヘルス（食材ストック等）
│   ├── sense_lab/          # Sense Lab
│   ├── users_settings/     # ユーザー設定
│   ├── db_viewer/          # DB 閲覧（管理系）
│   ├── dashboard/          # ダッシュボード付帯（記事トレーニング等）
│   ├── api/                # 共通 API 断片
│   └── assets/             # 静的アセット（JS/CSS/画像）
├── private/
│   ├── bootstrap.php       # Composer autoload + エラーハンドラ登録
│   ├── .env                # DB・API キー等（リポジトリに含めない想定）
│   ├── lib/                # Core（Auth, Database, Logger, BaseModel 等）
│   ├── apps/               # 機能別アプリ（App\ 名前空間）
│   ├── components/         # 共通 UI（sidebar, head_favicon 等）
│   └── vendor/             # Composer
├── migrations/             # SQL マイグレーション（done 等で整理）
└── docs/                   # 設計・手順ドキュメント
```

**ルーティング**: フレームワークのフロントコントローラは使わず、**各パスに `index.php` や個別 PHP を配置**する古典的な構成。API は多くが `*/api/*.php` 配下。

## 4. アーキテクチャ

### 4.1 ブートストラップとエラー処理

- `private/bootstrap.php` が `vendor/autoload.php` を読み込み、`Core\Bootstrap::registerErrorHandlers()` で PHP エラー・未捕捉例外・シャットダウンを `Core\Logger` に集約。
- `/api/` 配下などは JSON で 500 を返す分岐あり（通常画面は簡易 HTML エラー）。

### 4.2 認証・セッション・権限

- **ログイン**: `www/login.php` → `Core\Auth::login()`。`sys_users` の `password_verify`。
- **セッション内 `user`**: `id`, `id_name`, `role`, `apps`（サイドバー用ツリー）, `logo_text`, `default_route`, `sidebar_mode` 等。
- **ロール**: `sys_users.role` が `sys_roles.role_key` と対応（例: `admin`, `user`, `hinata`, `movie`, `hinata_admin` 等）。サイドバー表示文言は `sidebar.php` でロール別に分岐。
- **アプリマスタ**: `sys_apps`（親子・`route_prefix`・テーマ色・`default_route` 等）。ログイン時にツリー化してセッションへ格納（通常リクエストでは DB を叩かない方針）。
- **restricted モード**: `sys_roles.sidebar_mode = restricted` のとき `sys_role_apps` で許可アプリのみ表示。ダッシュボード非表示ロールは `www/index.php` で `default_route` へリダイレクトする処理あり。

### 4.3 データアクセス

- `Core\Database::connect()` が `private/.env` を読み、シングルトン PDO を返す。
- 各機能は `App\*\Model\*Model` と `Core\BaseModel` 派生で実装。ユーザー単位隔離が必要なモデルは `BaseModel::$isUserIsolated` 等で制御。

### 4.4 共通 UI

- **サイドバー**（`private/components/sidebar.php`）: セッションの `apps` ツリーから動的メニュー。テーマは Tailwind クラス名または `#` 始まりの HEX に対応。
- **テーマ**（`theme_from_session.php`）: 各画面で `app_key` に応じたヘッダー／カード色を統一。

## 5. 主要モジュールと機能（アプリ別）

以下は `private/apps/` と `www/` の対応から整理した**機能領域**です。実際のメニュー表示はロールと `sys_apps` の設定に依存します。

### 5.1 ダッシュボード（`www/index.php`）

- 日付・未完了タスク数、RSS 由来ウィジェット（好奇心ブースト・AI 関連・ブログ「パレオな男」）— `App\Dashboard\Service\DashboardFeedService`
- 記事を題材にしたトレーニング導線（`www/dashboard/`）
- 日向坂「次のイベント」・ミーグリ枠・推し誕生日、最優先タスク、Focus Note 未完了アクション数、クイックメモ（Note API へ保存）等

### 5.2 タスク管理（`TaskManager`）

- カテゴリ・優先度・期限付きタスク。API は `www/task_manager/api/`。

### 5.3 メモ（`Note`）

- シンプルなメモ一覧・保存。ダッシュボードのクイックメモからも利用。

### 5.4 Focus Note（`FocusNote`）

- 週次ページ、月次、ゴール設定、日次タスク、アクションと If-Then ルール等。

### 5.5 日向坂ポータル（`Hinata`）

- ポータル（ネタ・イベント・推し・リリース・ミーグリ・ブログ・トピック等）
- メンバー／イベント／楽曲／メディア（MV・イベント動画・一括登録）／ライブガイド／ミーグリ報告／カレンダー等
- 詳細は [日向坂ポータル_システム構成書.md](./日向坂ポータル_システム構成書.md)

### 5.6 映画（`Movie`）

- TMDB 連携による検索・インポート・タグ・ガチャ UI 等。`TMDB_API_KEY` が必要。

### 5.7 ドラマ（`Drama`）

- TMDB ベースの一覧・検索・インポート・タグ等。

### 5.8 アニメ（`Anime`）

- **Annict** OAuth（`ANNICT_CLIENT_ID` 等）。ダッシュボード／検索へのリンクは `.env` の `ANIME_BETA_ID_NAMES`（カンマ区切り `id_name`）でベータ制限あり。

### 5.9 ライブ遠征（`LiveTrip`）

- 遠征・行程・チェックリスト・経費・ホテル・目的地。Google Maps 系 API（Directions / Places / Static Map 等）を利用するサービスあり（`.env` のキー類）。

### 5.10 エンタメ（`Entame`）

- 視聴時間などの集計ダッシュボード（`/entame/`）。

### 5.11 ヘルス（`Health`）

- 例: 食材ストック（`kitchen_stock` 等）。

### 5.12 Sense Lab（`SenseLab`）

- エントリの作成・編集・表示（感覚・実験メモ用途）。

### 5.13 管理（`Admin`）

- アプリ／ロール設定、DB エクスポート・スキーマ閲覧、改善項目、ガイド、フレンドグループ等。管理者ロール前提の画面が多い。

### 5.14 その他

- **DB Viewer**（`db_viewer`）: スキーマ確認用。
- **ユーザー設定**（`users_settings`）: プロフィール・管理者によるユーザー／ロール操作 API。
- **友人アクティビティ**（`FriendsActivity`）: `www/friends_activity.php` 等。
- **API**（`www/api/guide.php`）: ガイド連携など。

## 6. 外部サービス・環境変数（代表例）

`private/.env` に記載する想定のものです（実際のキー名はコード参照）。

| 用途 | 変数例 |
|------|--------|
| DB | `DB_HOST`, `DB_NAME`, `DB_USER`, `DB_PASS`, `DB_CHARSET` |
| TMDB（映画・ドラマ） | `TMDB_API_KEY` |
| Annict（アニメ） | `ANNICT_CLIENT_ID`, `ANNICT_CLIENT_SECRET`, `ANNICT_REDIRECT_URI` |
| アニメ機能の利用者制限 | `ANIME_BETA_ID_NAMES` |
| YouTube / Maps 等 | 各 `App` 内のクライアントが参照 |

---

## 7. データベースとマイグレーション

- スキーマ変更用 SQL は `migrations/` 配下（完了分は `migrations/done/` 等に整理されている場合あり）。
- テーブル接頭辞の例: `sys_*`（ユーザー・ロール・アプリ）、`hn_*`（日向坂ドメイン）、`com_*`（共通メディア）、`lt_*`（ライブ遠征）等。

---

## 8. セキュリティ・運用上の注意（概要）

- パスワードは `password_hash` / `password_verify`。
- セッション Cookie は `secure` 前提（HTTPS 環境向け）。
- 管理画面・API はロールチェックを個別に実装しているため、**新規エンドポイント追加時は認可漏れに注意**。
