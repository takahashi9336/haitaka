# ダッシュボード UI/UX 改善提案

作成日: 2026-03-03

## 概要

メインダッシュボード (`www/index.php`) の UI/UX を改善するための提案をまとめたドキュメント。
現状の構成を活かしつつ、情報設計・操作性・視覚的な快適さを向上させることを目的とする。

---

## 現状のダッシュボード構成

| 順序 | セクション | 内容 |
|------|-----------|------|
| 1 | クイックメモ | Google Keep 風の簡易メモ入力（タイトル＋本文） |
| 2 | 日向坂 次のイベント | 直近のイベント表示（条件付き） |
| 3 | 最優先タスク | 優先度×期限でソートした最重要タスク（条件付き） |
| 4 | ナビゲーショングリッド | タスク管理・日向坂ポータル・Focus Note・管理画面への遷移カード |

---

## 改善提案一覧

### 1. ウェルカムセクションの追加

**優先度**: 中 / **難易度**: 低

メインコンテンツ最上部に、時間帯に応じた挨拶と日付を表示する。

```html
<div class="mb-8">
  <h2 class="text-2xl font-black text-slate-800">おはようございます、高梨さん</h2>
  <p class="text-sm text-slate-400 mt-1">2026年3月3日（火）・ 未完了タスク 5件</p>
</div>
```

- 時間帯ごとに挨拶を切り替え（おはよう / こんにちは / おつかれさまです）
- 未完了タスク数をサマリーとして表示
- ログイン直後の「自分のための画面」感を強化

---

### 2. クイックメモの折りたたみ化

**優先度**: 中 / **難易度**: 中

現状クイックメモは常に展開状態で、ダッシュボードの約 1/4 を占有している。

**改善案**:
- 初期状態: 1 行の入力バー（「メモを入力...」プレースホルダー）のみ表示
- クリック/フォーカス時にタイトル欄・アクションボタンが展開
- `Ctrl+Shift+N` のキーボードショートカットで即座にフォーカス

**効果**: ファーストビューでナビゲーションカードが見え、情報密度が向上する。

---

### 3. ナビゲーションカードの情報密度向上

**優先度**: 中 / **難易度**: 中

現状は各カードに数値 1 つとラベルのみ（例: 未完了タスク数「5」）。

**改善案**:
- プログレスバー追加: タスク完了率のミニバー（完了数/全体数）
- トレンドインジケーター: 前週比の増減を小さな矢印で表示
- サブ統計: カード下部に 2 つ目の小さな統計を表示（例: 今日期限のタスク件数）

```html
<div class="mt-3">
  <div class="flex justify-between text-[10px] text-slate-400 mb-1">
    <span>完了率</span><span>65%</span>
  </div>
  <div class="h-1.5 bg-slate-100 rounded-full overflow-hidden">
    <div class="h-full bg-indigo-500 rounded-full" style="width: 65%"></div>
  </div>
</div>
```

**効果**: カードを開かなくても状況を把握でき、判断速度が向上する。

---

### 4. 最優先タスクの UX 改善

**優先度**: 高 / **難易度**: 低

現状、期限超過時に `animate-pulse` で点滅し続けるが、視覚的なストレスが大きい。

**改善案**:
- `animate-pulse` を削除
- 代替: 左端に赤いアクセントバー（`border-l-4 border-red-500`）＋静的なグラデーション背景
- クイックアクション追加: カード右側に「完了にする」ボタンを配置
- 上位 2〜3 件のタスクをコンパクトなリストで表示する選択肢も検討

**効果**: 緊急性は十分伝わりつつ、画面のちらつきによる疲労を軽減できる。

---

### 5. Empty State のデザイン

**優先度**: 中 / **難易度**: 低

タスクが 0 件やネタが 0 件の場合、数値が「0」と表示されるだけで味気ない。

**改善案**:
```html
<!-- タスクが0件の場合 -->
<span class="text-3xl font-black text-emerald-500">
  <i class="fa-solid fa-check-circle"></i>
</span>
<p class="text-xs text-slate-400 mt-1">すべて完了！</p>
```

- 0 件の状態を「達成感」として演出する
- アイコンとポジティブなメッセージに切り替え

**効果**: 「ゼロ」がネガティブではなく達成感として伝わる。

---

### 6. 時間帯に応じた色合いの変化

**優先度**: 低 / **難易度**: 低

背景色が常に固定されている点を、微妙な色味の変化で改善する。

| 時間帯 | 背景のニュアンス |
|--------|-----------------|
| 朝 (5-11時) | ほんのりウォーム（`from-amber-50/30 to-slate-50`） |
| 昼 (11-17時) | 明るいニュートラル（現状維持） |
| 夜 (17-5時) | わずかにクール（`from-slate-50 to-indigo-50/20`） |

**効果**: 微妙な変化が「いつもの場所」感を壊さずに時間感覚を提供する。

---

### 7. レスポンシブの改善

**優先度**: 低 / **難易度**: 高

現状モバイルではカードが縦 1 列に並ぶだけ。

**改善案**:
- モバイルでのカード高さ削減: `h-44` → `h-auto md:h-44`
- スワイプ可能なカルーセル: モバイルではナビカードを横スワイプ対応
- 最優先タスクとイベントの固定表示: モバイルでスティッキー化

---

### 8. ヘッダー情報の整理

**優先度**: 高 / **難易度**: 低

ヘッダー右側に「ログインユーザー」ラベル＋ユーザー名のみが表示されている。

**改善案**:
- 「ログインユーザー」ラベルを削除（サイドバーに同じ情報があるため冗長）
- 代わりに今日の日付を `YYYY年M月D日（曜日）` 形式で表示
- ユーザー名はサブテキストとして小さく残す

**実装済み**: ヘッダーの日付表示 + ウェルカムセクションとして実装済み。

---

### 9. マイクロインタラクションの強化

**優先度**: 低 / **難易度**: 中

**改善案**:
- カードクリック時に軽いスケールダウン: `active:scale-[0.98]`
- 数値のカウントアップアニメーション: ページ読み込み時に 0 から実数値まで
- `alert()` を `App.toast()` に置換（`App.toast()` は `core.js` で既に実装済み）

特に `alert()` → `App.toast()` の置換は即効性が高い:
```javascript
// 変更前
alert('メモの内容を入力してください');

// 変更後
App.toast('メモの内容を入力してください');
```

---

### 10. セクション構成の再考

**優先度**: 高 / **難易度**: 中

ユーザーがダッシュボードを開く主目的は「今の状況を把握すること」。
メモ入力は二次的なアクションなので最上部にある必要はない。

**改善案（並び順）**:

| 順序 | セクション | 理由 |
|------|-----------|------|
| 1 | ウェルカム＋日付 | 最初に「今日」を把握 |
| 2 | 最優先タスク＋次のイベント | 「今日のアテンション」として横並び統合 |
| 3 | ナビゲーションカード | メインのアクション導線 |
| 4 | クイックメモ | 最下部またはフローティング FAB |
