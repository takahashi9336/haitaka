# live_trip UI設計書

## 1. 画面構成・レイアウト（共通）

- **共通**: `$appKey = 'live_trip'` で [theme_from_session.php](../../../private/components/theme_from_session.php)、[head_favicon.php](../../../private/components/head_favicon.php)、[sidebar.php](../../../private/components/sidebar.php)。詳細は [docs/common/31](../../common/31_共通UIコンポーネント.md)。
- **一覧・詳細のヘッダ**: `fa-plane` アイコン＋タイトル「遠征管理」。モバイルはハンバーガーでサイドバー（`www/assets/css/common.css` のレイアウトおよび `.cursor/rules/html-no-nested-anchors.mdc` のサイドバー規則）。
- **テーマ**: `sys_apps` で emerald系。詳細ヘッダ下で `:root { --lt-theme: <?= $themePrimaryHex ?> }` による `.lt-theme-btn` が使われる（`Views/index.php` 等）。
- **詳細ページ**: [`show.css`](../../../www/live_trip/css/show.css) を併用。マップ・タイムライン・チェックリスト等の **大きなブロック** は同一 HTML 内でセクション分割。

## 2. 画面別（要点）

### 遠征一覧（index.php）

- **レイアウト**: 日向坂ポータル相当の **最大幅（例: `max-w-5xl`）で中央寄せ**。
- **KPI カード（先頭）**: 今後の遠征件数・総遠征数・累計費用（費用＋交通＋宿泊の合算に準拠）。
- **期間トグル**: 「今後の遠征」「過去の遠征」「すべて」。**初期選択は「今後の遠征」**。見た目は詳細のタイムライン「予定追加」カードのカテゴリトグルに寄せる。
- **データ取得**: 初回 HTML 生成時に **今後／過去／日付未設定** のリストをまとめて用意し、トグル切替は **クライアント側で表示／非表示**（フルリロードしない）。URL の `period` は `history.replaceState` で同期。
- **「すべて」表示時**: 今後の遠征ブロック・過去の遠征ブロックを **帯見出し** で区切って表示。
- **ソート「新しい順」**: `TripPlanFilterService` により **今日に近い順**（今後は直近開催が上、過去は直近に終わったものが上、日付未設定は末尾）。
- **遠征カード**: 左アクセント線（状態色: 30日以内／今後／過去）、日付＋カウントダウン、イベント施設名（未紐付時は **メイン目的地の予定名** 等にフォールバック）、イベント種バッジ、費用サマリ等。スマホでは日付帯をカード上部に再配置。
- **イベント未紐付の基準日**: コントローラ側で **メイン目的地の `visit_date`** を `event_date` が空のときの並び・表示用日付として注入する場合あり。
- **しおり**: 一覧カードから **しおりボタンは出さない**（印刷用 `shiori.php` は別URL）。

### 遠征詳細（show.php）

- **構成**: ヒーロー（日程レンジ・残日数・会場リンク等）、**タブバー**（サマリ／参加情報／費用／宿泊／目的地／移動／タイムライン／チェックリスト）、各タブパネル、マップ（`live_trip_map.js` 等）。タイムラインと交通はマージ表示。
- **しおり**: ヒーローから **しおりボタンは出さない**。
- **Maps**: `UserPlaceModel` の `home`、目的地・会場座標の連携。API は `www/live_trip/api/*.php` 経由。
- **目的地タブ**:
  - 先頭に **KPI カード（件数・カテゴリ内訳・エリア）**。
  - **会場（イベント開催地）**は DB の `lt_destinations` ではなく、紐づけイベントから読み取り専用カードとして表示（編集不可）。
  - **複数イベントで会場が異なる場合**: 会場が **複数カード**で並ぶ（施設名＋住所の組が同一なら1枚に集約）。日向坂は `hn_event_place_address` / `hn_venue_latitude` 等、汎用イベントは `venue_address` / `venue_latitude` 等を参照。座標欠如時は Geocoding で会場ごとに補完しうる（APIキー有効時）。
  - 埋め込み地図の初期中心は **座標が取れた最初の会場**（なければ座標付き目的地などフォールバック）。
- **移動タブ**:
  - 上部 **KPI**（登録件数・交通費合計）と「＋移動を追加」導線。
  - **2カラム**（左: 日付別の登録済み移動一覧カード、右: 移動追加フォーム）。追加カード先頭は目的地追加と同様のアコーディオン見出し「＋ 移動を追加」。
  - 登録済み行では **交通機関**（`transport_type`）を緑系の文字色で強調、経路・所要・費用を併記。編集／削除はホバー時表示（目的地カードと同趣旨）。
  - 「Google Mapで検索」は所要時間入力近傍に配置。自宅登録は **移動追加カード右下** の折りたたみ。
- **タイムラインタブ**:
  - **ヘッダーKPIカード**: 目的地/費用タブのKPIカードと統一感を持つ3カラム構成。
    - **左**: `⏰ タイムライン`、件数（強調）＋日数、カテゴリ別件数バッジ（🚇移動/🎤ライブ/🛍️コラボ/🍱食事/🏨宿泊/📌その他）。
    - **中**: `活動時間（実働）`、合計時間（例: `⏱️ 合計 約 13時間`）、最早/最遅（例: `最早 10:00 ／ 最遅 23:00`）。
    - **右**: 表示切替（`リスト表示` / `日表示`）。デフォルトは**日表示**。
  - **日付フィルタカード**（KPIと「予定を追加」の間）:
    - 左に登録済み日付ボタン（昇順・横スクロール、相対日付/日付/予定件数の3段）。
    - 右に `全日まとめて表示`（表示切替ボタンと同じトグル見た目）。
    - デフォルト選択はイベント当日（イベント未紐付の場合は「ライブ扱い（メイン）」のある日を当日とみなす）。
    - 押下で日表示/リスト表示とも対象日付に絞り込み。
  - **予定を追加カード**:
    - 目的地追加カードと同じアコーディオンUI（`＋ 予定を追加`、展開/縮小）。
    - カテゴリはトグル風（🚇/🎤/🛍️/🍱/🏨/📌）。
    - 入力レイアウトは2行（1行目: 日付→時刻→所要→項目→追加、2行目: 場所→メモ）。
    - DBにカテゴリ列は無いため、現状は「項目（label）」に絵文字プレフィックスを付与して表現（実装準拠）。
  - **日表示（カレンダー）**:
    - 日ごとにカード化し、日付ヘッダー（例: `📅 4月4日（土） 当日`）＋サブ情報（予定件数/実働/会場など）を表示。
    - 本体は縦軸（縦線）＋カード型イベント。カテゴリ別に色・ドットを変える。
    - 予定間に一定以上の空きがある場合、「⋯ 空き時間（クリックで予定を追加）」要素を挿入し、クリックで追加フォームを展開して日付/時刻をプリセット。
  - **リスト表示**:
    - サマリタブの「行程プレビュー」カード（リストUI）を踏襲（タイトル文言・追加ボタンは非表示）。

### フォーム（create / edit）

- **form.php**:
  - `遠征タイトル`（必須）を入力して遠征プロジェクトを作成。
  - イベント選択（日向坂 / 汎用）は任意。未選択でも保存可能。
  - 保存後 `show` または一覧へ。

### イベント後追い紐付け（show / edit）

- **show.php（参加情報タブ）**: イベント未紐付時は空状態メッセージと「イベントを紐づける」導線を表示。
- **edit.php（form.php 再利用）**: 既存イベント紐付・追加・削除を実施。

## 3. 共通コンポーネントの利用（ライブ trip 固有）

| 参照 | 用途 |
|------|------|
| `theme_from_session` | `live_trip` のテーマ変数 |
| `flash_toast` | 成功・失敗メッセージ（該当画面で include される場合） |
| `App.toast` / `App.post`（[core.js](../../../www/assets/js/core.js)） | クライアント側操作フィードバック・JSON API |

## 4. 状態と表示制御

- **管理者以外**: コントローラで弾かれ、画面は原則表示されない。
- **データ未作成テーブル**: 一部 `try/catch` で空配列に落とす互換コードあり（マイグレーション前フォールバック）。

## 5. スタイル

- **Tailwind CDN** + 冒頭 `<style>` でフォント（Inter / Noto Sans JP）。カードの白背景・枠は `docs/common/32` のパターンに準拠。
