# 遠征管理機能改善対応 計画書

## 1. 背景と課題

- 新規登録時にイベント登録が起点になっており、導線が分かりづらい。
- 遠征管理を開始するために、先に予定（イベント）追加が必要になっている。
- 「遠征」そのものをプロジェクトとして扱う体験が弱い。

## 2. 改善方針

- 遠征作成時に `遠征タイトル` を必須入力とし、1遠征=1プロジェクトとして管理する。
- 遠征はイベント未紐付でも作成可能とする（イベント0件許容）。
- 遠征作成後に、既存イベントまたは新規汎用イベントを紐付け可能にする。
- 一覧・詳細画面は「遠征タイトル」中心の表示へ寄せる。

## 3. 仕様変更案

### 3.1 データモデル

- `lt_trip_plans` に `title` カラムを追加（必須）。
- 既存データは暫定値（例: 既存イベント名 / 「無題の遠征」）で移行する。
- マイグレーションは `private/migrations/todo` に追加する。

### 3.2 作成フロー

- `create.php` / `form.php`:
  - `遠征タイトル` 入力欄を追加。
  - イベント入力は任意ブロック化（未入力でも保存可）。
- `store.php`:
  - イベント0件でも `lt_trip_plans` を作成できるようにバリデーション変更。
  - 入力イベントがある場合のみ `lt_trip_plan_events` を作成。

### 3.3 作成後のイベント紐付け

- `show.php` または `edit.php` で「イベントを紐づける」導線を明示。
- 既存イベント選択と、汎用イベント新規作成導線の両方を提供。
- イベント未紐付時は空状態メッセージ＋CTAを表示。

### 3.4 表示改善（UI/UX）

- `index.php` のカードタイトルを `trip.title` 優先に変更。
- サブ情報としてイベント日程・会場を表示（未紐付時はプレースホルダ表示）。
- 詳細ヘッダも `trip.title` を主見出しにする。

## 4. 影響範囲

- **DB**: `lt_trip_plans`（title追加）、必要に応じて既存データ移行SQL。
- **Backend**: `LiveTripController`, `TripPlanModel`（create/update/get）。
- **Frontend**: `private/apps/LiveTrip/Views/form.php`, `index.php`, `show.php`。
- **Docs**:

  - `docs/live_trip/20_詳細設計/22_live_trip_ドメイン定義書.md`

## 5. 実施ステップ

1. DBマイグレーション作成（`title` 追加・既存データ補完）。
2. モデル/コントローラ更新（イベント0件作成許容、タイトル保存対応）。
3. 作成/一覧/詳細UI更新（導線改善・空状態表示）。
4. 設計書更新（基本/詳細/ADR反映）。
5. 動作確認（新規作成、未紐付保存、後紐付、既存編集への影響確認）。

## 6. レビュー観点

- イベント未紐付で遠征作成できるか。
- 遠征作成後にイベント紐付けできるか。
- 既存遠征データの表示崩れがないか。
- 一覧/詳細で遠征タイトルが主表示として機能しているか。
