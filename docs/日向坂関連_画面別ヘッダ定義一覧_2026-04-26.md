# 日向坂関連：画面別ヘッダ定義一覧

- 調査日: **2026-04-26**
- 対象: `www/hinata/*.php` から到達する **日向坂関連の画面（View）**
- 目的: 画面ごとにバラついているヘッダ（`<header>`）定義を棚卸しし、統一方針検討の材料にする

## 1. 調査方法（簡易）

- `www/hinata/*.php` の **エントリポイント** を起点に、呼び出している Controller/メソッドを確認
- Controller 内の `require_once __DIR__ . '/../Views/*.php'` から、対応する View を特定
- View 内の **`<header ...>` ブロック**を読み、主に以下を抽出
  - 高さ（`h-14` / `h-16`）
  - 背景（`bg-white` / `bg-white/80` + `backdrop-blur-md`）
  - 影（`shadow-sm` 等）
  - 戻る導線（`chevron-left` / `arrow-left` / `App.goBack`）
  - 右側アクション（管理導線、追加ボタン等）
  - 固定（`sticky top-0`）と `z-index`

## 2. 観測された「ヘッダの型（パターン）」まとめ

### 2.1 共通点（多くの画面）

- `components/sidebar.php` を読み込み、`main` 内に各画面独自の `<header>` を直書き
- `sticky top-0` を採用している画面が多い
- テーマ反映は `components/theme_from_session.php` に依存し、`$headerBorder`, `$headerIconBg`, `$headerShadow`, `$headerIconStyle`, `$cardIconBg` 等の変数を利用

### 2.2 パターンA: **h-14 / 白 / shadow-sm（通常ヘッダ）**

- 例: 楽曲、楽曲詳細、メンバー帳、イベント、ミーグリ、ネタ帳、ペンライト など
- 典型:
  - `class="h-14 bg-white border-b ... px-4 ... sticky top-0 ... shadow-sm"`
  - 左: モバイルメニュー + 戻る + アイコン + タイトル（`text-lg` 〜 `text-sm`）
  - 右: 管理導線 or 追加ボタンなど（小さめの pill ボタン）

### 2.3 パターンB: **h-16 / 半透明 + blur（ガラス風ヘッダ）**

- 例: ポータル、推し設定、動画一覧、動画管理（各種）、リリース管理、推し個別 など
- 典型:
  - `class="h-16 bg-white/80 backdrop-blur-md border-b ... px-6 ... sticky top-0"`
  - 右: `ポータルへ戻る` などのボタンを置く画面が多い

### 2.4 パターンC: **セットリスト系の簡易ヘッダ（独自）**

- `setlist_show.php`, `setlist_edit.php` は `$headerBorder` 等を使わず、`border-slate-100` 直指定かつ2段タイトル風
- 左: 戻る + 「セットリスト/セットリスト編集」ラベル + イベント名
- 右: 曲数や「編集」「保存」「追加」などの操作

## 4. ばらつきポイント（統一検討の観点）

- **高さ**: `h-14` と `h-16` が混在（見た目・情報密度・余白が変わる）
- **背景表現**: `bg-white` と `bg-white/80 + backdrop-blur-md` が混在（ガラス表現の有無）
- **戻るUI**: `chevron-left`/`arrow-left`/リンク先固定/`App.goBack` などが混在
- **タイトルの作り**: 1行タイトル/2行タイトル（セットリスト）、アイコンあり/なし（推し個別）
- **右側アクション**: ある画面・ない画面、ボタンスタイル（pill/角丸/色）も統一されていない
- **z-index**: `z-10` と `z-20` が混在（サイドバーやモーダルとの重なりが変わり得る）
