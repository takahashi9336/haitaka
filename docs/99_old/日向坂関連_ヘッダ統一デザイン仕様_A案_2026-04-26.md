# 日向坂関連：ヘッダ統一デザイン仕様（A案：白ベース）

- 調査/作成日: **2026-04-26**
- 対象: 日向坂関連画面（`/hinata/*`）のヘッダ（`<header>`）表現
- 目的: 画面ごとに異なるヘッダを、**白ベースの1つの型**に寄せて統一感と操作一貫性を担保する

## 1. デザイン方針（A案）

- **原則**: すべての画面でヘッダは「白・同じ高さ・同じ余白・同じ操作配置」を採用
- **狙い**: 業務ツールとしての視認性と、画面遷移時の“別アプリ感”の解消
- **非採用**: `bg-white/80` や `backdrop-blur-md` などのガラス表現は原則使わない（例外は後述）

## 2. ヘッダ基本コンポーネント仕様（Base Header）

### 2.1 サイズ/位置

- **高さ**: `h-14`（固定）
- **左右パディング**: `px-4`（固定）
- **境界線/影**:
  - 下線: `border-b` + テーマ変数（例: `$headerBorder`）
  - 影: `shadow-sm`
- **固定**: `sticky top-0`
- **z-index**: 画面共通で固定（推奨: `z-20` に統一）

### 2.2 レイアウト（3領域）

```
[Left: ナビ]                          [Right: アクション]  [Optional: Search]
 menu  + (optional back) + icon + title
 (optional subtitle)
----------------------------------------------
 (scroll content)
```

- **Left（必須）**
  - モバイルメニュー: `#mobileMenuBtn`（`md:hidden`）
  - 戻る: 任意（ルールは 3章、ポータルは原則無し）
  - アイコン: 24〜32px角の統一（例: `w-8 h-8 rounded-lg`）
  - タイトル: 1行（長い場合は `truncate`）
- **Right（任意）**
  - **制約なし**（複数配置OK）。ただし「見た目の統一（サイズ/角丸/タイポ）」は維持する
  - 目安として、Primary/Secondary の視覚差分（色・強調）は揃える
- **Search（任意）**
  - **日向坂ポータルのみ**、既存の検索ボックスを継続利用（PC幅で表示）

### 2.3 タイポ/情報量

- タイトル: `text-lg` + `font-black` を基本（密度が高い画面のみ `text-sm` を許容）
- サブタイトル（必要時のみ）:
  - 例: セットリストやミーグリレポの2行情報
  - `text-[10px] font-bold text-slate-400` 程度で統一

## 3. 戻る導線（Back）の統一ルール

### 3.0 日向坂ポータルは戻る不要

- **日向坂ポータル（`/hinata/index.php`）は戻るボタンを表示しない**

### 3.1 戻るは「上位に戻る」を基本

- **原則**: 戻るボタンは「セクション上位（一覧/ポータル）」へ戻す
- **例**:
  - 楽曲詳細 → 楽曲一覧（`/hinata/songs.php`）
  - リリース詳細 → 楽曲一覧（`/hinata/songs.php`）
  - ミーグリレポ → ミーグリ予定（`/hinata/meetgreet.php`）

### 3.2 `App.goBack()` は限定的に利用

- 履歴ベースの戻りが必須な画面（fromパラメータ等）だけに限定
- UIは同じアイコン（`chevron-left`）で統一し、挙動差を最小化

### 3.3 アイコン統一

- 戻るは `fa-chevron-left` に統一（`arrow-left` 等は原則廃止）

## 4. 右側アクション（Actions）の統一ルール

- **個数の制約なし（複数OK）**
  - 例: `+ 追加` / `保存` / `管理` / `編集` / `ポータルへ戻る` など
  - 表示条件（adminのみ等）は維持しつつ、見た目は共通化
- **ボタン見た目（推奨）**
  - pill: `rounded-full`
  - サイズ: `px-3 py-1.5`（または `px-4 py-2`）
  - 文字: `text-[10px] font-bold`（管理系）、追加/保存系は `text-xs font-bold`
  - 色: テーマに寄せる（`$cardIconBg` / `$cardIconText` など）

## 5. 例外扱い（A案でも崩して良いもの）

### 5.1 セットリスト（`setlist_show/edit`）

- 現状の2段タイトルは情報設計として妥当なため、**構造は維持**してOK
- ただし以下は統一する
  - 高さ: `h-14`
  - 戻るアイコン: `chevron-left`
  - z-index: 共通値
  - border/shadow: Base Header と同等

### 5.2 ミーグリレポ（アバター表示）

- アバターをヘッダに置く体験は差別化として有効
- ただし、余白/高さ/戻る/右側ボタンの寸法は Base Header に合わせる

### 5.3 推し個別（`oshi_member`）

- アイコン無しの“名前だけヘッダ”はブランディング的に成立し得る
- ただし、戻る/高さ/余白/右側の有無のルールは揃える

## 6. 実装方針（デザイン→実装へ落とすための提案）

### 6.1 「ヘッダの部品化」レベル

最小の統一でも効果が出る順:

1. **クラス規約の統一**（`h-14 bg-white border-b ... sticky ... shadow-sm z-20`）
2. **戻る/アイコン/タイトルの並び統一**
3. 共通 partial 化（例: `private/apps/Hinata/Views/partials/hinata_header.php`）して差分を引数化

### 6.2 引数として持たせたい情報（案）

- `title`（必須）
- `icon`（FontAwesome class / 省略可）
- `backHref`（上位戻りURL）
- `backMode`（`href` / `goBack`）
- `rightAction`（1つだけ：label/href/onclick/role）
- `variant`（例外用：`default`/`compact`/`twoLine`/`avatar`）

## 7. レビュー観点（対応方針レビュー用）

- **統一できるか**: `h-14` + 白ベースに全画面を寄せても違和感がないか
- **例外の妥当性**: セットリスト/ミーグリレポ/推し個別の例外範囲は適切か
- **戻るの意味**: “上位へ戻る”を基本にして迷いが減るか
- **右側操作の優先度**: Primaryが1つに絞れているか
