# エンタメ（アニメ・ドラマ・映画） UI設計書

## 1. 画面構成・レイアウト

### エンタメ統合ダッシュボード（`www/entame/index.php`）
- **構成要素**:
  - 共通サイドバー: `private/components/sidebar.php`
  - ヘッダー:
    - タイトル「エンタメ」
    - メディア切替リンク（映画/ドラマ/アニメ）: `/movie/index.php` `/drama/index.php` `/anime/index.php`
  - トータルサマリ（カード4枚）:
    - 見た（合計）
    - 見たい（合計）
    - 見てる（ドラマ＋アニメ）
    - 総視聴時間（映画＋ドラマ）
  - メディア別カード（3枚）:
    - 映画: 見た/見たい
    - ドラマ: 見た/見てる/見たい
    - アニメ: 見た/見てる/見たい
- **テーマ**:
  - `$appKey = 'entame'` で `theme_from_session.php` を通じて `--ent-theme` を設定し、`ent-theme-text` 等に反映。

### 映画ダッシュボード（`www/movie/index.php`）
- **構成要素（主要部）**:
  - 共通サイドバー
  - ヘッダー（映画）
  - スタッツカード（鑑賞本数/今月/見たい/視聴時間）
  - 検索（TMDB）: 入力 + 結果ドロップダウン（`MovieSearch`）
  - ガチャ: ローカルストレージ `mv_gacha` に当日回数/結果を保持（UI復元）
  - 友人が視聴した映画: `FriendsActivityService` 取得データから、プレビュー/追加ボタンを提供
  - おすすめ: personal/genre/trending のタブ相当（実装はセクション）
  - グラフ: 月別、ジャンル分布、評価分布（Chart.js）

### 映画一覧（`www/movie/list.php`）
- **構成要素（主要部）**:
  - タブ: `watchlist` / `watched`
  - 表示切替: grid/list（Cookie `mv_view_mode`）
  - 並び替え: `created_at`/`release_date`/`title`/（watched時のみ）`rating`/`watched_date`
  - リスト内フィルタ:
    - タイトルの部分一致
    - チップ（配信プロバイダ/タグ/ジャンル）
  - 追加導線:
    - インライン検索（TMDB）またはモーダル検索
    - TMDBに無い場合の手動追加（仮登録）

### ドラマダッシュボード/一覧（`www/drama/index.php`, `www/drama/list.php`）
- **構成要素（主要部）**:
  - 映画に準じる（TMDB検索、ガチャ、友人視聴、レコメンド、グラフ）
  - ドラマガチャ: ローカルストレージ `dr_gacha`、1日2回まで（UIで残回数表示）
  - 一覧表示切替: grid/list（Cookie `dr_view_mode`）

### アニメダッシュボード/一覧（`www/anime/index.php`, `www/anime/list.php`）
- **構成要素（主要部）**:
  - Annict OAuth 状態に応じた表示切替:
    - 未設定: 設定案内
    - 連携未完了: 連携CTA（authorize URL）
    - 連携済: スタッツ/検索/ガチャ/今期/友人視聴/グラフ
  - 検索: `AnimeSearch`（`/anime/api/search.php`）
  - ガチャ: `wanna_watch` からランダム（`/anime/api/gacha.php`）
  - 今期: `/anime/api/current_season.php` を読み込んで横スクロール表示（詳細へ遷移）

## 2. 共通コンポーネントの利用
- `private/components/sidebar.php`: サイドバー（モバイル時は `mobile-open` で開閉）
- `private/components/theme_from_session.php`: アプリ別テーマ変数取得・背景/ヘッダー等のクラス設定
- `private/components/head_favicon.php`: favicon 等の共通 head 要素
- JS共通: `/assets/js/core.js?v=2`（`App.post`, `App.toast` など）

## 3. 状態と表示制御 (State & Conditional Rendering)
- **統合ダッシュボード集計**:
  - 例外時は 0 埋め（`EntameController::dashboard()`）
- **アニメ（OAuth）**:
  - `oauthConfigured=false`: 設定不足の注意表示
  - `hasToken=false`: 連携CTA表示
  - `hasToken=true`: 機能フル表示 + 連携解除ボタン（`/anime/api/revoke.php`）
- **ガチャ（映画/ドラマ）**:
  - localStorage に当日状態がある場合は結果/回数制御を復元
  - 上限到達時は「本日のガチャは終了」を表示
- **友人視聴カード**:
  - 既に登録済みの場合はチェック表示（`_registered`）
  - 未登録の場合のみ、ステータス追加ボタンを表示

## 4. スタイル・デザインルール
- **テーマカラー**: `theme_from_session.php` 由来の `themePrimaryHex` を CSS 変数に反映（例: `--mv-theme`, `--dr-theme`, `--anime-theme`, `--ent-theme`）
- **共通UI傾向**:
  - カード: `bg-white rounded-xl border border-slate-100 shadow-sm`
  - ホバー: `hover:shadow-md` + `transform: translateY(-2px〜-4px)`
  - モバイルサイドバー: `@media (max-width: 768px)` で `transform: translateX(-100%)` を採用
